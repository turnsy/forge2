# Phase 4 — Vercel Sandbox + Python builder

**Goal:** Deterministic plan mutations via generated Python against a thin `forge_plan` library; sandbox writes `output/plan.json` validated on the server.

**Depends on:** Phase 1 (Sandbox SDK + credentials)

**Blocks:** Phase 3 E2E, Phase 6 QA

---

## Agent actions

### Python library (`forge-next/sandbox/forge_plan/`)

- [ ] Package layout: `__init__.py`, `plan.py`, `builders.py` aligned to schema v2.0.0
- [ ] Load seed from `/workspace/current_plan.json` (empty template if missing)
- [ ] Helpers (minimal v1 set — expand as needed):
  - `Plan.empty(name)` / `Plan.from_dict`
  - `add_week`, `add_day`, `add_exercise`, `add_set` (names TBD to match schema)
  - `to_dict()` → JSON-serializable matching `workout-plan.schema.json`
- [ ] Include `schema.json` copy or reference in sandbox workspace for generated code comments
- [ ] README in folder describing API for the codegen prompt

### Sandbox executor (`lib/sandbox/`)

- [ ] `createPlanSandbox({ currentPlan, normalizedContext, schema })`
- [ ] Write files into sandbox:
  - `current_plan.json`
  - `input_context.md` (or per-file `input_context/{name}.txt`)
  - `schema.json`
  - `forge_plan/` (library tree)
  - `run.py` (generated by LLM in Phase 3, or template entrypoint)
- [ ] Run Python; read `/output/plan.json` (ensure script writes there)
- [ ] `stop()` / dispose sandbox in `finally`
- [ ] Timeouts and error codes: `SANDBOX_TIMEOUT`, `SANDBOX_FAILED`, `MISSING_OUTPUT`
- [ ] Unit tests with mocked Sandbox client; one opt-in integration test gated by env `RUN_SANDBOX_INTEGRATION=1`

### Codegen prompt assets

- [ ] `lib/ai/plan-chat/python-codegen-prompt.ts` — constraints:
  - Only use `forge_plan` public API
  - Read `current_plan.json`, write `output/plan.json`
  - No network, no arbitrary imports
  - Keep script short

- [ ] Iteration: modified `current_plan.json` in → coherent `plan.json` out
