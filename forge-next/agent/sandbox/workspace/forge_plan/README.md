# forge_plan (Python)

Fluent builder library for workout plan mutations inside **Vercel Sandbox** (`python3.13` runtime).

Targets **workout-plan.schema.json v3.0.0**: `Plan → Week → Day → Block → Exercise → Set`.

Bundled into each sandbox run together with:

- `current_plan.json` — seed from client `currentArtifact` (server-only; not sent to the LLM)
- `run.py` — Python generated by AI Gateway

Output: `output/plan.json` validated server-side via `loadWorkoutPlan()`.

## Quick example

```python
from forge_plan import Day, Exercise, Plan, Week

plan = (
    Plan("1 week bench")
    .add_week(
        Week()
        .add_day(
            Day(name="Upper")
            .add_exercise(
                Exercise("Bench press").add_sets(reps=5, target=50, unit="kg", count=3)
            )
            .add_superset(
                Exercise("Bench press").add_sets(reps=10, target=30, unit="kg", count=3),
                Exercise("Incline bench").add_sets(reps=10, target=20, unit="kg", count=3),
            )
        )
    )
)

plan.week(0).day(0).block(0).exercise(0).set(0).update(reps=6, target=55, unit="kg")
plan.write_json("output/plan.json")
```

## Target values

- `target=50` → absolute load
- `target="75%"` → percentage load
- `unit` is always required (e.g. `"kg"`)
